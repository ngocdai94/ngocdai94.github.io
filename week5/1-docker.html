unRaid Operating System (OS) - Docker Containers

Dockers

From the unRaid post, I shared about unRaid Operating System, and its amazing features when it is using as a Networking Server and Network Attached Storage. Also, the unRaid’s ability to host multiple virtual machines to use as a software development environment, cyber security firewall to prevent hackers, and testing a malicious link from a stranger.

One of the other cool features from unRaid OS that I forgot to mention from the last post is Docker, which has been a huge success in the software development, and DevOps community.

What is Docker?

As stated from Wikipedia, Docker “is a set of platform-as-a-service (PaaS) products that use OS-level virtualization to deliver software in packages called containers. Containers are isolated from one another and bundle their own software, libraries and configuration files; they can communicate with each other through well-defined channels. All containers are run by a single operating-system kernel and are thus more lightweight than virtual machines”

In other word, Docker is similar to virtual machine. It requires a host machine to host a Docker container, which is similar to a disk International Organization for Standardization (ISO) image file. The host machine needs to install a container reader to read the Docker container images.

How to use Docker on unRaid?

Surprisingly, unRaid Operating System comes with pre-installed Docker. To use it, all you have to do is to navigate to the SETTING tab and then choose the Docker icon.


Next, make sure the selection box for Docker is enabled.


After that, you can navigate to the APPS tab to choose the built-in container for the unRaid OS, or you can import your own Docker container from the DOCKER tab.


What is the benefits of using Docker?

One of the main benefits of using Docker is that the host machine literally does not have to install any extra drivers to run Docker containers. Docker containers will handle all of the communication via its own channel of network using the Internet. Therefore, it allow fast deployment on multiple host machines. To help you understanding Docker easily, considering the example below.

Imaging an IT Technician has to configure and install softwares to more than 100 of PCs in a day. There is absolutely now way the IT Technician can finish this job in a day. However, if the IT Technician uses Docker containers on all of the PCs, all of them will connect to a central host, which the IT Technician send a command to all the PCs to install software on the Docker containers.
